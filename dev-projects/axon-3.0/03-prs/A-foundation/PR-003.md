## Spec deviation (recorded post-execution)

Phase 1 study claimed `shadow.py`, `document_parser.py`, `web_search.py`, `test_runner.py` should migrate to `_axon_paths`. On inspection: **none of them have workspace-relative defaults**:
- `shadow.py` requires `--shadow-dir` (no default)
- `document_parser.py` takes file paths from caller
- `web_search.py` has zero path defaults
- `test_runner.py` has zero workspace defaults

Phase 1 was wrong. No migrations needed.

**As executed**: 6 functional probes (compile-write, dispatch, usage, events, igap, axon-audit) added to `tools/health.py:PROBES`. Each runs in an isolated tmp workspace; each has a regression test in `TestNewFunctionalProbes`. Score 100/100, all 6 probes ACTIVE.

## Why

Today 12/50 tools haven't migrated to `_axon_paths`; 4 of those (`shadow`, `document_parser`, `web_search`, `test_runner`) really should ‚Äî they have workspace-resolving needs.

More urgent: 36/49 ACTIVE tools have only smoke probes (`--help` exits 0). Six of those are core to Phase D's orchestrator: compile-write, dispatch, usage, events, igap, axon-audit. Without functional probes, Phase C/D rides on hope.

## Implementation steps

1. **Path migration sweep** ‚Äî for each holdout, change argparse `default` to `default=default_workspace()` or `default=under_workspace("subpath")`. Add the import at the top, after the docstring.
2. **Probe: compile-write** ‚Äî fixture tiny program, run `compile-write --in <fixture> --out /tmp/x.cmp.md`, assert file exists + has expected header.
3. **Probe: dispatch** ‚Äî fixture index with one known program, run `dispatch match --query "<known>"`, assert program returned with confidence > 0.5.
4. **Probe: usage** ‚Äî `usage record --kind program --name probe-test --token-est 5`, then `usage top --window 1d`, assert "probe-test" in result.
5. **Probe: events** ‚Äî `events emit --event probe-test`, then `events listen --since 1m --filter probe-test`, assert event present.
6. **Probe: igap** ‚Äî `igap record --type fallback-exec --context probe --missing x --suggestion y`, then `igap report --days 1`, assert entry present.
7. **Probe: axon-audit** ‚Äî `axon-audit --dry-run`, assert structural keys (`integrity`, `coverage`, `recommendations`) present in output.
