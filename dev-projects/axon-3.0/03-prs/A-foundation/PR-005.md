## Spec deviation (recorded post-execution)

Phase 1 study claimed `compiler/COMPILER.md` Phase 3 was truncated at line 99. On read: line 99 is just the heading; the section has 6 complete rules (O1–O6). Phase 1 misread.

**Real gap** was missing rules for v2.3.0 ops (EVAL/RETRY/TEE). Closed by:
- 17 new grammar rules in GRAMMAR.md (6 EVAL + 6 RETRY + 5 TEE)
- Rule O7 (fuse EXEC+EVAL+IF+RETRY into RETRY-WITH-EVAL macro)
- Rule O8 (dedup TEE keys)
- Rule O9 (dead-store elimination — bonus opt rule from Phase 1)
- Rule O10 (redundant RETRIEVE collapsing — bonus opt rule from Phase 1)
- DEVELOPER.md changelog v1.1.3
- KERNEL-SLIM version stamp v1.1.2 → v1.1.3

No grammar test suite added — these are spec changes consumed by humans first, by the compile-optimizer next. PR-007 (compile pipeline consolidation) gets the integration tests when it builds the new unified `compile` tool.

## Why

LANG v2.3.0 defines 14 extensions but compiler grammar only maps the original 11. EVAL, RETRY, TEE programs today fall back to literal `EXEC("[description]")`, losing optimization. Half-spec compiler is a maintenance trap. Library-dev programs use these ops, so PR-004 depends on this.

## Implementation steps

(All axon/ writes require `L:dev-mode ≡ true`.)

1. **Open dev-mode**:
   ```
   python3 axon.py kv-store set --key L:dev-mode --value true
   ```

2. **GRAMMAR.md additions** — append three sections:
   - **EVAL section**:
     - "evaluate output against criteria" → `EVAL(output, {criteria})`
     - "check that output meets {tolerance}" → `EVAL(output, {criteria}, tolerance={tolerance})`
     - "verify result satisfies {predicate}" → `EVAL(result, {predicate})`
   - **RETRY section**:
     - "retry up to N times" → `RETRY(op, condition, max=N)`
     - "retry until {condition}" → `RETRY(op, condition={condition}, max=3)`
     - "retry on failure" → `RETRY(op, condition=failed, max=3)`
   - **TEE section**:
     - "save verbose output and summarize" → `TEE(result, {key}, summary=auto)`
     - "tee {output} into {key}" → `TEE({output}, {key})`
     - "store full result in {key}, return summary" → `TEE(result, {key}, summary=auto)`

3. **COMPILER.md Phase 3 (OPTIMIZE)** — replace truncated section with three concrete passes:
   - **Pass 3a — Dead-store elimination**: `STORE(W:k, v)` immediately followed by `CLEAR(W:k)` with no intervening RETRIEVE → both removed.
   - **Pass 3b — Redundant RETRIEVE collapsing**: consecutive `RETRIEVE(W:k)` where `k` is unchanged between → first call cached; subsequent are eliminated, replaced with literal value if known.
   - **Pass 3c — EXEC inlining**: `EXEC(program)` where the target program is ≤3 ops AND has no LOOP/UNTIL → inline the body.

   Each pass produces a numbered diagnostic; emit `OPT-{pass}-{id}: {action}` to compiler stderr.

4. **Tests** — `tests/test_compiler_grammar.py::TestEvalRetryTee` with one test per new rule, asserting the natural-language input compiles to the expected op.

5. **Dev-mode close**:
   ```
   python3 axon.py kv-store set --key L:dev-mode --value false
   ```

## Test strategy

- `TestEvalRetryTee` covers each new grammar rule.
- Compile a fixture program containing all three ops in natural language; assert output `.cmp.md` contains the symbolic ops, not literal `EXEC("...")`.
- `compile-optimizer test-all` after PR-004 lands → all library-dev programs compile cleanly.

## Rollback plan

Revert the GRAMMAR.md and COMPILER.md edits via git. All authored programs that relied on the new rules either fall back to literal EXEC (no functional break, just suboptimal) or stay broken until reverted.
