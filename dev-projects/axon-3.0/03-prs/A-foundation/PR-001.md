# PR-001 — Foundation fixes

**Phase:** A — Foundation
**Complexity:** S
**Status:** not-started
**Depends on:** none

## Why

Four bite-sized bugs blocking other work:
- `cron status` is unreadable from the CLI (TypeError on every reboot)
- Session resume restores zero state because the snapshot writer produces 0 lines
- HOWTO points at `KERNEL.md` which was archived 2026-05-02
- KERNEL-SLIM header (v1.1.0) disagrees with DEVELOPER.md changelog (latest entry v1.1.1)

None of these survive a CI gate but none are caught by it either. They surface as confusion every reboot.

## Affected files

| File | Change |
|---|---|
| `tools/cron.py` | Fix line 165 TypeError on `data["jobs"]` (data is a list, not a dict) |
| `tools/session_save.py` | Stop writing empty snapshot when state collection fails; either write a real snapshot or skip with LOG(WARN) |
| `axon/HOWTO.md` | `KERNEL.md` → `KERNEL-SLIM.md` (every occurrence) |
| `axon/KERNEL-SLIM.md` line 2 | Bump version stamp `v1.1.0` → `v1.1.1` to match DEVELOPER.md |
| `axon/DEVELOPER.md` | Append `[2026-MM-DD] v1.1.2 — Phase 3 axon-3.0 work begins` to changelog |
| `tests/test_tools_kernel.py` | New `TestCron::test_check_does_not_raise` |

## Implementation steps

1. **cron.py fix** — read `tools/cron.py:165`. The current code assumes `data["jobs"]` but `cron.json` may be a list-shaped store. Switch to defensive: `jobs = data["jobs"] if isinstance(data, dict) else data`.
2. **Add cron regression test** — `TestCron::test_check_does_not_raise` runs `python3 tools/cron.py check` from a tmp_ws with an empty cron.json, asserts rc=0 and JSON output has `overdue` key.
3. **session_save.py snapshot** — current code writes the snapshot file unconditionally; make it only write when there's actual content (`bool(snapshot["keys"])`). Add LOG(WARN, "session-save: no W: keys to snapshot") when empty.
4. **HOWTO rewrite** — grep for `KERNEL.md` in HOWTO.md, replace with `KERNEL-SLIM.md`. Verify boot chain references stay consistent.
5. **Version stamp sync** — KERNEL-SLIM line 2: `v1.1.0` → `v1.1.1`. axon/ writes require dev-mode.
6. **Changelog entry** — DEVELOPER.md changelog: add v1.1.2 entry stamped today, citing PR-001.

## Test strategy

- `TestCron::test_check_does_not_raise` — verifies the fix.
- `TestSessionSave::test_skips_empty_snapshot` — fixture session with no W: keys, assert no file written or empty file logged.
- `tools/lint_paths.py` — must remain clean.
- `python3 tools/health.py` — must report 100/100.

## Rollback plan

Each change is small and reversible via git revert. The KERNEL-SLIM version bump can be reverted without breaking any consumer (no code reads the version stamp).

## Acceptance criteria

- ✓ `python3 tools/cron.py check` returns valid JSON with `overdue` key (no traceback)
- ✓ `TestCron::test_check_does_not_raise` passes
- ✓ session-save no longer writes empty snapshots
- ✓ HOWTO.md references KERNEL-SLIM.md exclusively (grep -c "KERNEL.md" returns 0 outside archive/)
- ✓ KERNEL-SLIM line 2 version matches DEVELOPER.md changelog head
- ✓ Health 100/100, lint clean

## Risks

- KERNEL-SLIM edit requires `L:dev-mode ≡ true`. Must enable + restate command + disable on completion (Core Rule 9 + no-queue rule).
- session-save change might surface that no programs ever populate snapshot meaningfully — that's PR-009's concern; here we just stop writing the empty file.

## Out of scope

- Actually populating the snapshot with useful state (PR-009).
- Restoring state on resume (PR-009).
- Auto-firing cron (PR-010).
