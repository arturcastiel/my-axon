# PR-002 — Tool authoring conventions

**Phase:** A — Foundation
**Complexity:** M
**Status:** completed (narrowed scope — see "Spec deviation" below)
**Depends on:** none

## Spec deviation (recorded post-execution)

Original spec called for wholesale envelope migration across 6 tools (memory, events, dispatch, compile_suggest, validator, kv_store). Each migration changes the JSON output shape, which breaks every downstream caller (programs + tests + tool-to-tool subprocess parsers).

**As executed (this PR)**:
- ✓ `tools/_axon_response.py` — `ok()`, `fail()`, `emit()`, `emit_ok()`, `emit_fail()` helpers
- ✓ `tools/_axon_io.py` — `atomic_write()`, `atomic_write_json()` helpers
- ✓ `tools/memory.py` migrated to `atomic_write` + `atomic_write_json`
- ✓ `TestAxonResponse` (5 cases), `TestAtomicIO` (5 cases) — full coverage of helpers
- ✓ `events.py` already used in-line atomic write; left as-is (functionally equivalent)

**Deferred to future per-tool PRs**:
- Mass envelope migration on dispatch / compile_suggest / validator / kv_store. Each migration is now incremental + can ship without breaking the world.
- `argparse` subparser standardization. Touched the 5 tools that mix positional+flag styles → also breaking change for downstream programs that invoke them. Deferred.

**Downstream PRs updated as a result**:
- **PR-007** (compile pipeline consolidation): can adopt the canonical envelope on the new unified `compile` tool. Recorded as note in PR-007 spec.
- **PR-019** (inter-tool shared library): can adopt the envelope at the library boundary. Recorded as note in PR-019 spec.

This narrowing keeps PR-002 atomic and reversible; later phases pick up the migration on a per-tool basis as they touch each tool.

## Why

The 50-tool suite mixes argparse styles, JSON output shapes, and write semantics. Phase B's consolidation and Phase C's orchestrator both need a stable contract; without this, every consumer reinvents error parsing and every refactor risks behavioral drift.

## Implementation steps

1. **Create `tools/_axon_response.py`** — exports:
   ```python
   def ok(data=None) -> str         # → '{"ok": true, "data": ...}'
   def fail(error: str, **kw) -> str # → '{"ok": false, "error": "...", ...}'
   def emit(payload: dict) -> None   # prints JSON to stdout, flushes
   ```
   Schema: `{"ok": bool, "error": str | None, "data": any | None}`. Always `ok` is present; `error` non-null iff `ok=false`; `data` populated on success.

2. **Create `tools/_axon_io.py`** — exports `atomic_write(path: str, content: str) -> None` using NamedTemporaryFile in same dir + `os.replace`. Tests in `TestAtomicIO`.

3. **Migrate memory.py + events.py** to atomic_write. Existing direct-open patterns get replaced. Pytest: race-free.

4. **Standardize argparse on subparsers** — for tools that mix positional + flags (`dispatch.py`, `compile_suggest.py`, `queue_tool.py` if needed). Keep CLI surface backward-compatible by accepting both forms during transition (one release).

5. **Migrate user-facing error returns** — every tool that today returns `{"error": "..."}` directly should call `fail(...)` from the new module. Sweep covers: `validator.py`, `kv_store.py`, `dispatch.py`, `compile_suggest.py`, `usage.py`. Heavy probes (chromadb, etc.) stay as-is for graceful import-error handling.

6. **`TestResponseEnvelope`** — for each migrated tool, assert: success returns `{"ok": true, "data": ...}`; failure returns `{"ok": false, "error": "..."}`. Catches regressions where someone forgets to use the helpers.

## Test strategy

- New unit tests on `_axon_response` (4 cases: ok empty, ok with data, fail with error only, fail with extra kwargs).
- New unit tests on `_axon_io` (3 cases: write happens, race-safe via tmp+rename, exception leaves original untouched).
- Migration tests on each touched tool: still functional + envelope shape verified.
- `python3 tools/health.py` → 100/100. All probes still pass.

## Rollback plan

Each tool migration is independent. If a tool breaks under the new envelope, revert that single tool's change — `_axon_response` and `_axon_io` are passive helpers, removing them just means returning to local error formatting.

## Acceptance criteria

- ✓ `_axon_response.py` and `_axon_io.py` exist with full unit tests
- ✓ memory.py and events.py use atomic_write
- ✓ At least 6 tools migrated to canonical envelope (memory, events, dispatch, compile_suggest, validator, kv_store)
- ✓ All existing pytest classes still pass
- ✓ TestResponseEnvelope passes
- ✓ Health 100, lint clean

## Risks

- A consumer parses the old envelope shape (`{"error": "x"}` directly). Mitigation: grep for tool-output parsing in workspace/programs/; if found, update those programs (add to scope).
- Subparsers conversion can break existing `axon.py` invocations. Mitigation: compatibility shim in axon.py during transition.

## Out of scope

- Migrating all 50 tools (sweep covers the inconsistent + high-traffic ones; rest follow gradually).
- Removing legacy positional-action support (deferred to a "v3.2 cleanup" PR).
