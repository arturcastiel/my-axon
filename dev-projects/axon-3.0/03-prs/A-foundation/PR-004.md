**Phase:** A — Foundation

## Spec deviation (recorded post-execution)

Compiled all 13 targets (9 library-dev + igap-improve + code-dev-pr-review + code-dev-shadow + show-memory bonus). Forward-coverage regression test added (`TestForwardCompilationCoverage`).

**Pre-existing issues surfaced (not introduced by this PR; out of scope to fix here)**:
- `TOOL(shell, ...)` not in REGISTRY.json — affects code-dev-pr, code-dev-pr-spec, library-dev-search, etc. Verify.py blocks. Pre-existing — happens for code-dev-pr too. Tracked separately; "shell" needs registering as a built-in or these programs need to use named tools.
- 3 existing compiled programs (`code-dev-study.cmp.md`, `resume.cmp.md`, `turn-log.cmp.md`) lack the `# COMPILED:` header (they're authored as plain programs in `compiled/`). Pre-existing; not introduced here.
- Library-dev family includes 4 `!STUB — Phase 2` programs (cite, intersect, report, search) that don't have full structure (priority flag, DONE() call, ## OUTPUT). Compilation succeeded; structural validation flags them as incomplete. By design — these are placeholders pending real implementation.
- `test_compiled_token_ratio` fails for the new programs because the current "compile" pipeline is mostly copy-with-header — no real optimizer exists yet (PR-005 spec'd the rules; implementation is future work). This is a real limitation surfaced by PR-004.

**Verdict**: 13 .cmp.md created, 13 + dispatch entries via compile-write. Forward-coverage test green. Pre-existing failures unchanged. Library-dev compile gap closed structurally; semantic fixes (TOOL(shell) registration, !STUB → real impl, real optimizer) tracked as follow-ups outside this Phase.

Library-dev family is 0/9 compiled while code-dev is 10/11. Largest visible coverage gap. Phase D's auto-compile rule needs a clean baseline. A regression test makes the gap permanent-fixed.

## Implementation steps

1. **Run compiler on the 12 targets**:
   ```
   for prog in library-dev library-dev-new library-dev-ingest library-dev-explain library-dev-intersect library-dev-search library-dev-report library-dev-cite library-dev-status igap-improve code-dev-pr-review code-dev-shadow ; do
     python3 axon.py compile-optimizer compile --program $prog
   done
   ```
2. **Verify each via test-all** — `python3 axon.py compile-optimizer test-all`. Expect ALL_PASS or document any false-positive heading mismatches (the `code-dev-pr-spec` style issue from Phase 1 study).
3. **Update dispatch-index.json** — `compile-write` already appends to dispatch-index; confirm 12 new entries.
4. **Regression test** — `tests/test_compiled_regression.py::TestCompiledCoverage::test_every_program_compiled` lists `workspace/programs/*.md`, filters those with `# PROGRAM:` header, asserts `compiled/<name>.cmp.md` exists. Allowlist: reference docs (faq, glossary, versions, authoring-guide), one-shots (migrate-workspace, my-axon-init).
5. **Update menu** — surface `code-dev-pr-review` as a top-level code-dev subcommand.

## Test strategy

- All 12 targets compile cleanly.
- `compile-optimizer test-all` verdict is ALL_PASS (or 1 expected false positive documented).
- New regression test passes; deliberately fail it by removing one .cmp.md → verify red.
- Dispatch can match queries to the new compiled programs (smoke: `dispatch match --query "ingest pdf"` → library-dev-ingest).

## Rollback plan

`rm workspace/programs/compiled/{the 12 new files}` and revert dispatch-index entries. Regression test removed.
