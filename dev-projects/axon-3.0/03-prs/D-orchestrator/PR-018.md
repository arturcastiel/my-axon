## Carryover from PR-009/10/11/12 execution

**Menu state already partially populated** — PR-009/10/11/12 added several boot-JSON fields the menu can consume directly:

- `cron.tick`        ← PR-010 (last tick result)
- `cron.seeded`      ← PR-011 (first-boot seed status, null thereafter)
- `cron.overdue_count` ← PR-001/010 (jobs awaiting action)
- `last_snapshot`    ← PR-009 (`{ts, key_count, w_keys}`)
- Drift state        ← PR-012 (already added via menu badge in PR-012)

The "Self-improvement" panel in PR-018 should pull from these existing fields rather than recomputing. Concretely:

```
SELF-IMPROVEMENT
─────────────────────────────────────────────
  Drift         {boot.cron.drift.state}      ← already in menu (PR-012)
  Cron          {N} jobs · last: {prog} {Δt} ← from boot.cron.tick.ran
  Pending       {boot.cron.overdue_count} overdue
  Snapshot      {boot.last_snapshot.key_count} W: keys captured at {ts}
  Audit         {N} entries since last review · auto-actions
  Auto-mode     {ON|OFF} (toggle: kv-store set L:auto-improve true)
```

PR-012's drift badge stays as-is (compact line in OS STATE); PR-018 adds the richer "Self-improvement" panel below it. No conflict.

**Output-layer footer drift wiring** — deferred from PR-012 to here (per PR-012's spec deviation). PR-018 should touch `axon/OUTPUT-LAYER.md` under dev-mode to add the drift line. Small change; the menu badge already proves the gate API is solid.
