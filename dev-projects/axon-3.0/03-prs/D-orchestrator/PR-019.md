# PR-019 — Inter-tool shared library

**Phase:** D — Orchestrator + UX
**Complexity:** L
**Status:** not-started
**Depends on:** PR-002, PR-007

## Carryover from Phase A/B execution

**From PR-002 (narrowed)**: The library boundary is the natural choke point to enforce the canonical envelope. Functions in `tools/_axon_lib.py` should always return `dict` (not raw JSON strings) shaped per `_axon_response.ok()`/`fail()`. Callers that previously parsed ad-hoc JSON shapes get migrated to the envelope as part of this PR's refactor.

**From PR-007 (deferred deliberately to here)**: PR-007 built `tools/compile.py` as a subprocess dispatcher (`compile <sub>` → `subprocess.run([backing_script, ...])`). This was chosen to ship the unified UX without forcing an in-process refactor of the three legacy scripts. **PR-019 owns the elimination of those subprocess hops**:

- `compile.py` should grow `from _axon_lib import compile_format, compile_rank, compile_scan, compile_verify, ...` and call them directly.
- The three legacy scripts (`compile-write.py`, `compile_suggest.py`, `compile_optimizer.py`) should expose `main(argv=None)` callable functions that `_axon_lib` wraps.
- The deprecation shims for those legacy tools (currently still on disk as OPTIONAL in REGISTRY) can route to the same library entry points instead of subprocessing back to themselves.
- Net effect: the same functional surface, but `compile rank` runs in-process instead of fork+JSON-parse.

This deferred refactor is the largest single perf win in the project; the orchestrator (PR-017) calls `compile rank` daily, so eliminating ~50ms per hop matters at scale.

**From PR-006**: `fire_hook` already uses `AXON_ROOT` for tool resolution; when PR-019 lifts it into `_axon_lib`, the same anchoring approach (resolve via `_axon_paths.AXON_ROOT`) carries over.

## Why

Subprocess hops cost ~50ms each and serialize JSON. With 5 hops in axon-audit alone, that's a quarter-second wall on every audit. Library imports remove the cost without changing the user surface.

## Implementation steps

1. **Identify subprocess hop hotspots** — grep for `subprocess.run` invoking other tools. Target list (from Phase 1 study):
   - `compile_suggest.py` line 38: shells to compile-write
   - `axon_audit.py` lines 80–110: multiple tool subprocesses
   - `dispatch_stats.py` line 75: shells to dispatch
   - `compile_optimizer.py`: shells to test, simulate, compile-write
   - Potentially: probe_kv_store, probe_dispatch in health.py

2. **Create `tools/_axon_lib.py`** — exports:
   ```python
   def compile_format(prog_path, out_path) -> dict
   def compile_rank(window, threshold) -> dict
   def dispatch_match(query) -> dict
   def usage_record(kind, name, **kw) -> dict
   def usage_top(window) -> dict
   def events_emit(event, payload) -> dict
   def kv_get(key) -> dict
   def kv_set(key, value) -> dict
   def memory_get(scope, key) -> dict
   def memory_set(scope, key, value) -> dict
   ```
   Each is a thin wrapper around the underlying tool's main logic, called as a function.

3. **Refactor callers** — replace `subprocess.run(["python3", "tools/compile-write.py", ...])` with `from _axon_lib import compile_format; result = compile_format(...)`.

4. **Keep CLI surface** — every tool still has its own CLI; library imports are an internal optimization.

5. **Tests**:
   - `test_compile_format_lib_equivalent_to_cli` — call both, assert same result
   - `test_dispatch_match_lib_equivalent` — same
   - `test_axon_audit_runs_under_threshold_ms` — timing benchmark; should be ≥3x faster

## Test strategy

- Pytest equivalence tests (lib vs CLI produce same result).
- Benchmark: time `axon-audit` before/after; expect ≥3x speedup.

## Rollback plan

Revert callers to subprocess. Delete `_axon_lib.py`. CLI surface unchanged so external consumers unaffected.

## Acceptance criteria

- ✓ `_axon_lib.py` exists with at least 6 functions
- ✓ At least 3 callers refactored (compile_suggest, axon_audit, dispatch_stats)
- ✓ Equivalence tests pass
- ✓ axon-audit runs ≥3x faster
- ✓ Health 100, lint clean

## Risks

- Coupling: a bug in library affects all callers. Mitigation: library functions are thin; main logic stays in tools/X.py.
- Module import side effects (e.g. argparse parsing on import) need to be guarded by `if __name__ == "__main__":` blocks. Audit each.

## Out of scope

- Refactoring all subprocess hops in the suite (target the 5 hottest).
- Removing the CLI surface (must stay for users + downstream programs).
