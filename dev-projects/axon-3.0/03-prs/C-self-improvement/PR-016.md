# PR-016 — Rollback for auto-edits

**Phase:** C — Self-improvement plumbing
**Complexity:** M
**Status:** not-started
**Depends on:** PR-009

## Carryover from Phase A/B execution

- `_axon_io.atomic_write` (PR-002) — use for rollback snapshot writes so partial writes can't corrupt rollback history. memory.py was already migrated; extend the same pattern to compiled-program rollbacks here.
- `tools/memory.py` already had a 3-version L: rollback pattern; PR-002 didn't change the algorithm, just made writes atomic. Lifting the pattern to compiled programs + W: keys is mechanical.

## Carryover from PR-009/10/11/12 execution

- **PR-009 snapshot is rollback-eligible**: `last-session-snapshot.md` captures W: state at session-end. PR-016's W: rollback can use the same JSON shape — restore via `session_save restore` is essentially "rollback to the snapshot". Worth aligning so the same code path serves both manual restore and orchestrator rollback.
- **Cron `last_run`/`run_count` rollback isn't needed**: PR-010's cron tick increments these on success. Rollback would imply "un-ran" a job — not meaningful for the kinds of auto-actions cron triggers (igap report, compile rank, etc. are read-mostly). Skip cron-state rollback.
- **PR-012 drift trace**: a diverged trace can be reset (`drift reset`). That's a destructive action — worth a rollback snapshot of the prior trace JSON before resetting. Adds another rollback target alongside compiled programs and W: keys.
- **Atomic write helper from PR-002**: snapshot writes to the rollback dir should use `atomic_write_json`. The existing memory.py rollback pattern uses direct `json.dump`; if PR-016 lifts it into a shared helper, switch that to atomic too (defensive against concurrent restore + new auto-edit).

## Why

Auto-edits without rollback is high-risk. With it, the worst case is "ran an unwanted optimization, ran undo, fine". Three-version cap keeps storage bounded.

## Implementation steps

1. **Generalize the 3-version pattern** — extract from `memory.py` into a helper function `snapshot_with_cap(target_path, cap=3)`:
   - Before any auto-edit, snapshot the file to `<target>.rollback/<basename>-<ts>.<ext>`.
   - Keep the last 3; older snapshots are deleted in FIFO order.

2. **undo.py CLI**:
   - `undo --target <path>` — restore most recent snapshot
   - `undo --target <path> --version N` — restore Nth back (1=most recent, 3=oldest available)
   - `undo --list <path>` — list available rollback snapshots
   - `undo --target W:<key>` — special syntax for W: keys (translates to memory.py rollback)

3. **Hook into auto-edit paths**:
   - Compiled program rewrite: orchestrator (PR-017) calls `snapshot_with_cap(workspace/programs/compiled/X.cmp.md)` before `compile format`.
   - W: key change: orchestrator snapshots via memory.py before STORE.

4. **Tests**:
   - `test_compiled_rollback` — snapshot, modify, undo, assert restored.
   - `test_w_key_rollback` — STORE, snapshot, STORE different value, undo, assert restored.
   - `test_three_version_cap` — 5 snapshots, assert only last 3 remain on disk.
   - `test_undo_list` — N snapshots, list returns N entries with timestamps.

## Test strategy

- Pytest covers all paths.
- Smoke: take a compiled program, modify, undo, diff.

## Rollback plan

Revert undo.py + memory.py changes. Existing `.rollback/` directories stay (no harm).

## Acceptance criteria

- ✓ undo.py supports compiled programs and W: keys
- ✓ 3-version cap enforced
- ✓ Tests pass
- ✓ Existing memory.py L: rollback still works (regression)
- ✓ Health 100, lint clean

## Risks

- Disk usage if many rollback dirs accumulate. Mitigation: cap at 3; old ones deleted.
- Concurrent edits could race the snapshot. Mitigation: use atomic_write helper from PR-002.
- W: key snapshots leak sensitive intermediate state. Mitigation: stays in my-axon/ which is private.

## Out of scope

- Rollback of E: episodic entries (append-only by design; rollback complicates audit).
- Rollback of axon/ kernel files (write-gate already requires dev-mode; rollback would be a separate concern).
- Cross-rollback (undo a chain of related edits). Phase D might want this; deferred.
