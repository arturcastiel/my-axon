# PR-013 ‚Äî Prompt-log opt-in flow

**Phase:** C ‚Äî Self-improvement plumbing
**Complexity:** S

## Why

Prompt-log is hard-disabled today. Auto-enabling is wrong; never offering is also wrong. One-time clear consent + easy off-switch.

## Implementation steps

1. **prompt-log-consent.md** ‚Äî new program. Outline:
   ```
   PROGRAM: prompt-log-consent
   desc: One-time consent flow for prompt logging.

   Step 1: Show consent text (what gets logged, where, retention, opt-out).
   Step 2: QUERY user: yes / no / details.
     - If "details": show the full schema + example log line, then re-query.
     - If "yes": STORE(L:prompt-log-enabled, true), STORE(L:prompt-log-consent-asked, true).
     - If "no": STORE(L:prompt-log-enabled, false), STORE(L:prompt-log-consent-asked, true).
   Step 3: Confirm + show how to disable later (`prompt-log disable`) and purge (`prompt-log purge`).
   ```

2. **Menu integration** ‚Äî workspace/programs/menu.md adds:
   ```
   IF L:prompt-log-consent-asked ‚â¢ true ‚Üí
     ‚óà Prompt logging not yet configured ‚Äî run: prompt-log-consent
   ```

3. **prompt_log.py purge** ‚Äî new subcommand `prompt-log purge --days <N>` removes entries older than N days. Default purges everything.

4. **Auto-purge in memory-compact** ‚Äî when `memory-compact` runs, also purge prompt-log entries older than 90 days.

5. **Tests**:
   - `test_consent_yes_enables_logging`
   - `test_consent_no_disables_logging`
   - `test_consent_idempotent` (asked once, doesn't re-ask)
   - `test_purge_removes_old_entries`

## Test strategy

- Pytest covers consent state transitions.
- Smoke: run consent program manually, verify each path (yes/no/details).

## Rollback plan

Delete the program; revert prompt_log.py purge subcommand. Consent state persists via L: keys; revert sets nothing destructive.

## Acceptance criteria

## Risks

- Surfacing a privacy prompt might surprise users on first boot post-update. Mitigation: phrasing makes opt-in clear; opt-out is one command.
- Purge is destructive. Mitigation: confirmation prompt unless `--force`.

## Out of scope

- Encrypting the prompt log (out of scope for solo-dev).
- Cross-machine sync of consent state (consent is per-host).
