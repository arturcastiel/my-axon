# PR-015 — Audit trail for auto-edits

**Phase:** C — Self-improvement plumbing
**Complexity:** M
**Status:** not-started
**Depends on:** PR-001 + PR-002

## Carryover from Phase A/B execution

- `_axon_response.ok()` / `fail()` and `_axon_io.atomic_write_json()` (PR-002) — use these for `auto_audit.py`'s output and append-only writes. Canonical shape from day one.
- Audit entries' `actor` field should distinguish: `orchestrator` · `cron` (the daemon firing PR-010 jobs) · `hook` (events tool's `hook-fire` from PR-006). The hook actor is real now — fire_hook works from any cwd post-PR-006 fix.
- Audit `target` field for compile auto-actions should record both old and new tool names (`compile format` not `compile-write`), since PR-007 made `compile` the canonical name. The old script names still resolve via deprecation-shim path; audit prefers the canonical form.

## Carryover from PR-009/10/11/12 execution

- **`actor` field — `cron` is real**: PR-010's `cron tick` and PR-011's first-boot autowire produce auto-actions. The audit trail should distinguish `cron` (boot-time tick) from `cron-manual` (explicit `cron run`). Rate-limit + auto-toggle context belongs in the entry's metadata.
- **`actor` field — `drift-halt`**: PR-012's R_DRIFT_GATE rule blocks output when drift is diverged. Each block is a defensible auto-action (the system refused something) — worth recording. Schema addition: `actor: "drift-halt"` with `target: <program>` and `gate_passed: false` (the gate explicitly didn't pass, that's the point).
- **`target` for compile actions**: PR-007 made `compile` canonical. Audit entries should store `tools/compile.py {subcommand}` not the deprecated `tools/compile-write.py` etc.
- **session-save audit entries**: PR-009's `session_save restore` is technically an auto-edit (rewrites `<ws>/memory/working/*.md`). When the orchestrator (PR-017) triggers restore, that path should be audited. Manual user-triggered `session_save` doesn't need auditing — distinguish via `actor`.
- **Per-day audit file location**: `my-axon/log/auto-edits/YYYY-MM-DD.md` matches the existing `igap` log layout from PR-010's seeded job (`igap report --days 7`). Same structure aids cross-reading; a single weekly `igap improve` cron can scan both.

## Why

Trust is the gate to enabling autonomy. Without an audit trail, the user can't verify what happened overnight; without verification, autonomy stays off. This PR is the trust foundation for PR-017 (the orchestrator).

## Implementation steps

1. **auto_audit.py** with subcommands:
   - `auto-audit record --actor <orchestrator|cron|hook> --action <op> --target <path> --before-hash <h> --after-hash <h> --before-excerpt <s> --after-excerpt <s> --gate-passed <gate> --rule-invoked <rule>`
     Appends to `my-axon/log/auto-edits/YYYY-MM-DD.md` (markdown table format).
   - `auto-audit list [--days N] [--actor X]` — list recent entries.
   - `auto-audit summary [--days N]` — group by action type, count, total impact.

2. **Schema** (each entry):
   ```
   | timestamp | actor | action | target | before_hash | after_hash | gate | rule |
   ```
   Plus a sub-row with before/after excerpts (≤120 chars each, truncated with …).

3. **auto-actions program** — workspace/programs/auto-actions.md:
   - Reads `auto-audit summary --days 7`.
   - Renders a table.
   - Surfaces unread count via `L:auto-actions-last-viewed-ts`.
   - On exit, updates the last-viewed timestamp.

4. **Menu integration** — workspace/programs/menu.md:
   ```
   IF unread auto-edits > 0 →
     ▶ {N} auto-edits since last review — type: auto-actions
   ```

5. **Tests**:
   - `test_record_appends` — record 3 entries, assert file has 3 rows
   - `test_summary_groups` — record 5 entries (3 compile, 2 tune), assert summary shows {"compile": 3, "tune": 2}
   - `test_query_by_date` — record entries on 2 different days, query single day, assert filter

## Test strategy

- Pytest covers core mechanics.
- Smoke: manually invoke `auto-audit record`, verify file structure.

## Rollback plan

Delete auto_audit.py + auto-actions.md. The audit log files in my-axon/log/auto-edits/ stay (can be inspected manually).

## Acceptance criteria

- ✓ auto-audit tool functional with record/list/summary
- ✓ auto-actions program shows recent activity
- ✓ Menu surfaces unread count
- ✓ Tests pass
- ✓ Health 100, lint clean

## Risks

- Audit log can grow unbounded if heavy auto-edit traffic. Mitigation: memory-compact runs monthly per PR-011 cron and archives entries > 90d.
- Hash mismatches if files are rewritten externally between record and verify. Mitigation: hashes are point-in-time at the moment of the edit; not used for later verification (just diagnostic).

## Out of scope

- Cryptographic signing of audit entries (overkill for solo-dev).
- Sync to remote (audit log stays local in my-axon/).
