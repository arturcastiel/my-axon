# PR-007 — Compile pipeline consolidation

**Phase:** B — Tool consolidation
**Complexity:** M
**Status:** completed (subprocess-hop elimination deferred to PR-019)
**Depends on:** PR-002, PR-005

## Note from PR-002 narrowing

PR-002 deferred mass envelope migration. When this PR builds the unified `compile` tool, it should adopt the canonical envelope (`tools/_axon_response.py`'s `ok()`/`fail()`/`emit()`) for all output. This makes the new tool a clean reference implementation; consumers can migrate to the envelope as they re-integrate.

## Why

Three tools (`compile-write`, `compile-suggest`, `compile-optimizer`) for three phases of one pipeline. Subcommands match user mental model and remove the subprocess hops that today serialize JSON between tools.

## Implementation steps

1. **Create `tools/compile.py`** — argparse with subparsers:
   - `compile format --in <prog> --out <cmp>` (was compile-write)
   - `compile rank --window 7d --threshold 5` (was compile-suggest)
   - `compile optimize --program <name>` (was compile-optimizer compile)
   - `compile verify --program <name>` (was compile-optimizer verify)
   - `compile test-all` (was compile-optimizer test-all)
   - `compile check-composition` (was compile-optimizer check-composition)

2. **Move old files** to `_compile_*.py` (underscore prefix = "internal, no CLI"). Each becomes a Python module exposing `format()`, `rank()`, `optimize()`, etc. functions. The new `compile.py` imports them and dispatches.

3. **Removes the subprocess hop** — today `compile-suggest` shells out to `compile-write`; with shared imports, it becomes a function call. Faster, no JSON serialize/parse.

4. **Aliases in axon.py**:
   ```python
   ALIASES["compile-write"]      = "compile format"
   ALIASES["compile-suggest"]    = "compile rank"
   ALIASES["compile-optimizer"]  = "compile"   # subcommand passed through
   ```
   For subcommand mapping, `axon.py` learns to parse "compile-optimizer test-all" → "compile test-all".

5. **Sweep workspace/programs/** — replace `TOOL(compile-write, ...)`, `TOOL(compile-suggest, ...)`, `TOOL(compile-optimizer, ...)` with `TOOL(compile, format ...)`, `TOOL(compile, rank ...)`, `TOOL(compile, optimize ...)`.

6. **Probes** — `health.py:PROBES` updates the keys; functional probes from PR-003 stay working.

7. **Tests** — rewrite `TestCompileOptimizer` as `TestCompile`, add subcommand-shape tests.

## Test strategy

- All compile-optimizer pytest cases still pass (just tool name changed).
- Backward compat: `axon.py compile-suggest suggest` still routes correctly via alias.
- Performance: compile-suggest → compile-format flow no longer subprocess-hops; spot-check timing.

## Rollback plan

Revert compile.py + the `_compile_*.py` renames. Restore compile-write.py / compile_suggest.py / compile_optimizer.py from git. Programs that switched to new tool names need their references reverted too. Tests revert.

## Out of scope

- Removing the alias compat layer (separate cleanup PR later).

## Spec deviation (recorded post-execution)

Original spec called for renaming `compile-write.py` → `_compile_format.py`, `compile_suggest.py` → `_compile_rank.py`, `compile_optimizer.py` → `_compile_optimize.py` and importing the underscore modules in-process from the new `compile.py`.

**Executed differently**: kept the three legacy scripts in place as-is and built `compile.py` as a subprocess dispatcher that routes subcommands to them. Reasons:
- Renaming would break direct invocations from existing programs and any external tooling.
- Importing the legacy scripts as modules requires each to be made importable (their `if __name__ == "__main__":` blocks need rework). That's a bigger refactor.
- The user-visible benefit (single `compile <sub>` CLI) is delivered without the internal churn.

**Subprocess-hop elimination is deferred to PR-019** (shared library). PR-019's `tools/_axon_lib.py` will expose `compile_format()`, `compile_rank()`, etc. as Python functions, and `compile.py` will import them. This keeps the win incremental.
