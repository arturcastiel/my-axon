## Implementation steps

1. **save-workspace ⊕ workspace-backup**:
   - `workspace-backup` already supports setup/push/status. `save-workspace` is essentially `workspace-backup push` with a different name. Delete `save-workspace.md`.
   - Update menu hint that referenced "save-workspace" to point at "workspace-backup push".
   - Update axon.py command table if save-workspace appears there.

2. **code-dev-pr ⊕ code-dev-pr-spec**:
   - `code-dev-pr-spec` is OPM-flavoured spec writer. Move its template into `workspace/templates/code-dev-pr-opm.tpl.md`.
   - `code-dev-pr` gains `--style {default,opm,detailed}` flag (default = `default`).
   - When `--style opm`, code-dev-pr loads the OPM template instead of default.
   - Delete `code-dev-pr-spec.md`.
   - Update code-dev dispatcher: `code-dev pr-spec` becomes `code-dev pr --style opm`.

3. **Recompile** affected programs (workspace-backup, code-dev-pr, code-dev).

4. **Update help / menu** to reflect new flags.

5. **Tests** — pytest assertion that:
   - `save-workspace.md` doesn't exist
   - `code-dev pr --style opm` runs and produces an OPM-flavoured output
   - `code-dev pr --style default` still works the original way
