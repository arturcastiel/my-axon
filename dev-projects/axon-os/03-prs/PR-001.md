## `axon/KERNEL-SLIM.md` changes

### 1. BOOT STEPS — step 2: add `my-axon` detection block

After `TOOL(boot)` + `TOOL(prefs)` and after G-10 workspace path validation, add:

```
   # my-axon detection — fires after workspace path validated
   myaxon-path ← RETRIEVE(W:myaxon-path) |
     TOOL(shell, "git -C {W:ws-os} rev-parse --show-toplevel 2>/dev/null") + "/my-axon"
   myaxon-md   ← "{myaxon-path}/MYAXON.md"
   IF FILE-EXISTS(myaxon-md) →
     EXEC(READ(myaxon-md))   ← loads all W:myaxon-* STORE ops
     LOG(INFO, "boot: my-axon loaded — {myaxon-path}")
   ELSE →
     → "  ⚠  No my-axon/ user-data folder found at {myaxon-path}"
     answer ← QUERY(user): "[F]resh  [C]lone existing repo  [S]kip  (default: F)"
     answer ← answer | "F"
     IF answer ≡ "F" OR answer ≡ "" →
       STORE(W:my-axon-init-mode, "fresh")
       EXEC(my-axon-init)
     IF answer ≡ "C" →
       STORE(W:my-axon-init-mode, "clone")
       EXEC(my-axon-init)
     IF answer ≡ "S" →
       LOG(WARN, "boot: my-axon absent — limited mode, no data persistence")
```

### 2. Memory scopes — add `W:myaxon-*` definitions

In the memory scopes section, add alongside `W:` / `L:` / `E:` scopes:

```
**my-axon scope** — `W:myaxon-*`: user runtime data, absolute paths from MYAXON.md.
  W:myaxon-path         ← absolute path to my-axon/ root
  W:myaxon-name         ← user name/identifier
  W:myaxon-dev-projects ← my-axon/dev-projects/
  W:myaxon-memory       ← my-axon/memory/
  W:myaxon-longterm     ← my-axon/memory/longterm/
  W:myaxon-episodic     ← my-axon/memory/episodic/
  W:myaxon-working      ← my-axon/memory/working/
  W:myaxon-local        ← my-axon/memory/local/ (gitignored, machine-specific)
  W:myaxon-log          ← my-axon/log/entries/
  W:myaxon-igap         ← my-axon/log/igap/
  W:myaxon-turns        ← my-axon/log/turns/
  W:myaxon-chats        ← my-axon/chats/
  W:myaxon-plans        ← my-axon/plans/
  W:myaxon-libraries    ← my-axon/libraries/
  W:myaxon-generated    ← my-axon/generated/
All W:myaxon-* keys are loaded from MYAXON.md at boot.
```

### 3. Layer architecture description — update

```
Layer 1  axon/                  OS core — write-gated by dev-mode, never by programs
Layer 2  workspace/             AXON config — programs, preferences, addons, tools
```
