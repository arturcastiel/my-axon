## New purpose

The old `migrate-workspace.md` targeted moving `workspace/` to a sibling directory — that design is obsolete. The new version migrates existing user data from `workspace/` into a freshly-scaffolded `my-axon/`, then dissolves the nested `.git` so `workspace/` is re-absorbed into `axon.git`.

---

## New `migrate-workspace.md` program

```

# desc:    One-time migration: move user data from workspace/ into my-axon/,
#          dissolve workspace/.git, re-track workspace/ in axon.git
# usage:   migrate-workspace
# WARNING: Outputs shell commands. Human executes each step.

!NORM | SPAWNED → RUNNING

ASSERT(L:cognition-frame ≡ "AXON-OS") | HALT("Identity lost — run: boot axon")
STORE(W:active-program, "migrate-workspace")

    "my-axon/ already exists at {myaxon-path}.

    "workspace/ has no .git — nothing to dissolve. May already be migrated.")

→ "  Source:  {ws-abs}"

→ ""
```
